# KRON SIEM v1.0 — Release Notes

**Release date:** TBD (Phase 5 Gate)
**Status:** Release Candidate
**Classification:** Public

---

## Overview

KRON v1.0 is the first Generally Available release of the KRON SIEM platform —
a production-grade, on-premise security information and event management system
built for the Indian market.

Key highlights:
- 50,000+ events per second ingestion pipeline (ClickHouse + Redpanda)
- SIGMA-based detection with ONNX ML anomaly scoring
- Native Indian compliance: CERT-In 2022, DPDP Act, RBI IS Audit, SEBI CSCRF
- MSSP multi-tenant portal with 4-gate tenant isolation
- Flutter mobile app for iOS and Android (SOC analyst on the go)
- WhatsApp + email P1 alert delivery in < 2 minutes p99

---

## What's New

### Core Platform

#### Event Pipeline
- **kron-agent**: eBPF (Linux) and ETW (Windows) native kernel telemetry collection
- **kron-collector**: Agentless syslog (UDP/TCP RFC 3164/5424), HTTP intake, cloud connectors
- **kron-normalizer**: CEF, LEEF, JSON, syslog format detection; 15+ timestamp formats; GeoIP enrichment
- **kron-stream**: SIGMA rule matching, ONNX isolation forest anomaly scoring, IOC enrichment
- **kron-storage**: Adaptive storage — ClickHouse for hot data, DuckDB for dev/edge deployments

#### Detection & Response
- **SIGMA rules**: 500+ built-in rules (upstream corpus + KRON India Pack)
- **SOAR playbooks**: Automated response with analyst biometric approval gate
- **Risk scoring**: Composite 0–100 score combining rule severity, anomaly score, IOC hits, asset criticality
- **kron-alert**: WhatsApp Business API + SMTP delivery with per-tenant routing and escalation

#### Compliance Engine
- **CERT-In Directions 2022**: All 13 incident categories mapped and reportable
- **DPDP Act 2023**: 72-hour breach notification deadline tracking with personal data access trail
- **RBI IS Audit**: Privileged access audit trail + data localisation verification
- **SEBI CSCRF**: MTTD/MTTA/MTTR metrics with SLA breach tracking
- Evidence packages: signed ZIP with manifest, events, alerts, audit log

#### MSSP Portal
- Multi-tenant management with per-tenant configuration (WhatsApp number, language, frameworks)
- 4-gate tenant isolation: JWT claims → query rewrite → storage layer → bus topic separation
- Continuous canary test verifying isolation every 5 minutes
- Tenant onboarding wizard and offboarding with audit trail preservation

### Web UI (SolidJS)
- Dashboard: real-time alert feed, EPS meter, MITRE heatmap, asset risk gauge
- Alert queue with keyboard navigation (J/K navigation, A to acknowledge)
- Event search with 1B+ row ClickHouse queries in < 3 seconds
- Rule builder: SIGMA YAML editor with live syntax validation
- Compliance dashboard: report generation, evidence ZIP download
- MSSP portal: tenant management with framework multi-select

### Mobile App (Flutter)
- iOS 14+ and Android 5.0+ (API 21+)
- Biometric login (Face ID / fingerprint) + TOTP MFA
- Real-time P1/P2 push notifications via Firebase Cloud Messaging
- Alert feed with severity filtering and pull-to-refresh
- Alert detail with acknowledge / resolve / false-positive actions
- SOAR approval screen with mandatory biometric confirmation
- On-call schedule with page-analyst capability

### Infrastructure
- Helm chart for Kubernetes (production deployment)
- Docker Compose for single-node deployments
- USB installer for air-gapped environments
- One-line installer for Ubuntu 20.04/22.04 and RHEL 9
- musl-linked static binaries for x86_64 and aarch64
- All release artifacts signed with cosign + GPG

## Security

- All binaries cosign-signed; `cosign verify-blob` command in release page
- CycloneDX SBOM (JSON + XML) published with every release
- `checksums.sha256` file GPG-signed by KRON release key
- Dependencies audited with `cargo audit` on every CI run
- Pentest completed against staging environment prior to GA

**Verify a binary:**
```bash
cosign verify-blob \
  --key https://raw.githubusercontent.com/Hardik364/Kron/main/cosign.pub \
  --signature kron-collector-x86_64-unknown-linux-musl.sig \
  kron-collector-x86_64-unknown-linux-musl
```

## Upgrading

v1.0 is the first GA release — no upgrade path from pre-release builds.
For beta/RC users, perform a fresh install and restore from backup:

```bash
# On new node
curl -fsSL https://install.kron.security | bash -s -- --version 1.0.0

# Restore data
./scripts/restore.sh --backup-file kron-backup-YYYYMMDD.tar.gz
```

## Known Limitations

| ID | Description | Workaround | Target |
|---|---|---|---|
| KL-001 | eBPF agent requires kernel 5.4+ with BTF enabled | Use agentless syslog collection on older kernels | v1.1 |
| KL-002 | DuckDB backend limited to single-node (no cluster) | Use ClickHouse backend for production | v1.1 |
| KL-003 | ONNX model trained on generic baseline; India-specific training data pending | Tune anomaly threshold per tenant in config | v1.1 |
| KL-004 | Mobile app requires internet for Firebase push notifications | Fall back to polling mode in fully air-gapped environments | v1.2 |
