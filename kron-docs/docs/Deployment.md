# KRON — Deployment Guide

**Version:** 1.0

---

## Deployment Methods

| Method | Tier | Time to running |
|---|---|---|
| USB stick (air-gap) | Nano, Standard | <15 minutes |
| One-line installer (internet) | Nano, Standard | <10 minutes |
| Helm chart | Standard, Enterprise | <30 minutes |
| Terraform + Helm | Enterprise | <2 hours |

## Nano Deployment

### Prerequisites
- x86_64 machine (physical or VM)
- 8 GB RAM minimum (16 GB recommended)
- 250 GB disk minimum
- Ubuntu 22.04 LTS or RHEL 9 (dedicated — nothing else runs on this machine)
- Internet access (or USB installer for air-gap)

### One-line Install (Internet)
```bash
curl -fsSL https://install.kron.security | sudo bash
```

The installer:
1. Detects OS and hardware
2. Installs Docker if not present
3. Pulls KRON Nano images
4. Runs setup wizard (org name, admin password, WhatsApp number)
5. Starts all services via Docker Compose
6. Prints access URL and initial credentials

### USB Install (Air-gap)
```bash
# On any internet-connected machine:
curl -fsSL https://releases.kron.security/nano/latest.iso -o kron-nano.iso
# Write to USB (8 GB minimum):
sudo dd if=kron-nano.iso of=/dev/sdX bs=4M status=progress

# On target machine:
# Boot from USB → KRON installer runs automatically
# Follow CLI wizard
# Reboot → KRON running on internal IP
```

### Docker Compose Structure
```yaml
# /opt/kron/docker-compose.yml (auto-generated by installer)
services:
  kron-server:
    image: kron/server-nano:1.0.0
    restart: unless-stopped
    volumes:
      - /var/lib/kron:/data
      - /var/log/kron:/logs
    ports:
      - "443:443"    # Web UI + API
      - "8883:8883"  # Agent ingestion
    environment:
      - KRON_MODE=nano
      - KRON_ORG=${ORG_NAME}
    cap_drop: [ALL]
    cap_add: [NET_BIND_SERVICE]
    security_opt: [no-new-privileges:true]

  kron-agent-manager:
    image: kron/agent-manager:1.0.0
    restart: unless-stopped
    volumes:
      - /var/lib/kron/agents:/agents
    network_mode: host  # required for fleet management

  minIO:
    image: minio/minio:latest
    restart: unless-stopped
    volumes:
      - /var/lib/kron/cold:/data
    command: server /data
    environment:
      - MINIO_ROOT_USER=${MINIO_USER}
      - MINIO_ROOT_PASSWORD=${MINIO_PASSWORD}
```

### Post-Install: Deploy Agents
```bash
# On KRON server — generate agent deployment token
kron-ctl agent-token create --name "linux-fleet" --ttl 24h
# → outputs: TOKEN=eyJ...

# On target Linux hosts:
curl -fsSL https://kron-server/install-agent | sudo bash -s -- --token TOKEN --server kron-server:8883

# Ansible (bulk deployment):
ansible-playbook -i inventory.yml kron-agent.yml --extra-vars "kron_token=TOKEN kron_server=kron-server:8883"

# Windows (PowerShell):
Invoke-WebRequest https://kron-server/install-agent.ps1 -OutFile install.ps1
.\install.ps1 -Token TOKEN -Server kron-server:8883
```

## Standard Deployment

### Prerequisites
- 1–2 dedicated servers (physical or VM)
- 16–32 GB RAM per server
- 2 TB NVMe SSD minimum
- Ubuntu 22.04 LTS or RHEL 9
- Network: servers can reach each other on all ports (internal network)

### Install k3s + KRON
```bash
# On server 1 (primary):
curl -sfL https://get.k3s.io | sudo sh -

# Install KRON via Helm:
helm repo add kron https://charts.kron.security
helm repo update

helm install kron kron/kron \
  --namespace kron \
  --create-namespace \
  --values values-standard.yaml \
  --wait \
  --timeout 10m
```

### values-standard.yaml
```yaml
global:

      cpu: "16"

redpanda:
  mode: single
  persistence:
    size: 500Gi

streamProcessor:
  replicas: 3
  resources:
    requests:
      memory: "2Gi"
      cpu: "2"

queryApi:
  replicas: 2
  ingress:
    enabled: true
    host: kron.acmebank.internal
    tls: true

ai:
  enabled: true
  mode: cpu  # or "gpu" if GPU node present
  mistral:
    enabled: true

  whatsapp:

```

### Verify Deployment
```bash
kubectl get pods -n kron
# All pods should be Running

kubectl exec -n kron deploy/kron-query-api -- kron-ctl health
# → {"status":"healthy","clickhouse":"ok","redpanda":"ok","agents":0}

# Access web UI:
# https://kron.acmebank.internal
# Default admin: admin@kron.local / (set during install)
```

## Enterprise Deployment

### Prerequisites
- 3+ physical servers (or VMs across availability zones)
- Minimum per data node: 128 GB RAM, 32 cores, 10 TB NVMe
- Minimum per compute node: 64 GB RAM, 16 cores, 500 GB SSD
- Optional GPU node: 32 GB RAM, NVIDIA A10G or RTX 4090
- 25 Gbps network between nodes
- Kubernetes 1.28+

### Step 1: Infrastructure Provisioning (Terraform)
```bash
git clone https://github.com/kron-security/kron-deploy
cd kron-deploy/terraform/baremetal

# Edit terraform.tfvars with your server IPs and specs
terraform init
terraform apply
# → provisions: k8s cluster, storage volumes, network config, DNS
```

### Step 2: Install KRON Enterprise
```bash
helm install kron kron/kron \
  --namespace kron \
  --create-namespace \
  --values values-enterprise.yaml \
  --values values-secrets.yaml \
  --wait \
  --timeout 30m
```

### values-enterprise.yaml (excerpt)
```yaml
global:
  tier: enterprise

clickhouse:
  mode: sharded-ha

  replicas: 2  # per shard
  keeper:
    replicas: 3

    requests:
      memory: "96Gi"

redpanda:
  mode: ha-cluster
  replicas: 3
  replicationFactor: 3
  resources:
    requests:
      memory: "32Gi"
      cpu: "8"

spire:
  enabled: true
  server:
    replicas: 3

cilium:
  enabled: true

    enabled: true

  enabled: true
  mode: ha-raft
  replicas: 3
  hsm:
    enabled: true
    pkcs11LibPath: "/usr/lib/softhsm/libsofthsm2.so"

  enabled: true

  enabled: true

  enabled: true

  mistral:
    enabled: true

      kron.security/gpu: "true"
```

### Step 3: Validate Enterprise HA
```bash
# Test Redpanda cluster
kubectl exec -n kron redpanda-0 -- rpk cluster health
# → All brokers healthy, leader elected

# Test ClickHouse replication
kubectl exec -n kron clickhouse-0 -- clickhouse-client -q "SELECT * FROM system.replicas WHERE is_leader = 1"

# Test SPIRE
kubectl exec -n kron spire-server-0 -- spire-server bundle show

# Test Vault HA
kubectl exec -n kron vault-0 -- vault status
# → HA Enabled: true, Active Node: vault-0

# Run HA failover test
kubectl delete pod -n kron clickhouse-shard-0-replica-0
# → Traffic should transparently route to replica-1
# → No query failures in API

# Run cross-tenant isolation test
kron-ctl test tenant-isolation
# → PASSED: 0 cross-tenant rows returned
```

## Air-Gap Operations

For fully disconnected environments:

### Offline Image Mirror
```bash
# On internet-connected machine:
kron-ctl package create-bundle --version 1.0.0 --output kron-bundle-1.0.0.tar

# Transfer bundle to air-gapped network via approved method (USB, secure transfer)

# On air-gapped machine:
kron-ctl package import kron-bundle-1.0.0.tar
# → Loads all container images into local registry
```

### Offline Threat Feed Updates
```bash
# On internet machine:
kron-ctl feeds snapshot --output kron-feeds-$(date +%Y%m%d).tar

# Transfer via USB

# On KRON server:
kron-ctl feeds import kron-feeds-20260101.tar
# → Updates IOC bloom filter, MISP data, GeoIP
# → Verifies feed integrity (signed snapshot)
```

### Offline ONNX Model Updates
```bash
# On internet machine:
kron-ctl models download --version latest --output kron-models-1.0.tar

# Transfer + import same as above
kron-ctl models import kron-models-1.0.tar
# → Hot-reloads new models without service restart
```

## Upgrade Procedure

### Nano (Docker Compose)
```bash
# Pull new version
kron-ctl upgrade --version 1.1.0

# What it does:
# 1. Creates backup of current data
# 2. Pulls new images
# 3. Applies schema migrations (zero-downtime for additive changes)
# 4. Rolls over services one by one
# 5. Verifies health before completing
```

### Standard/Enterprise (Helm)
```bash
helm upgrade kron kron/kron \
  --namespace kron \
  --values values-standard.yaml \
  --version 1.1.0 \
  --wait \
  --atomic  # rolls back automatically if upgrade fails
```

Schema migrations run as a Kubernetes Job before pods are updated. Migrations are always backward-compatible (additive) — old pods continue running while migrations apply.

## Monitoring the Deployment

### Grafana Dashboards (pre-installed)
- **KRON Overview** — EPS, alert rate, agent count, query latency
- **ClickHouse Performance** — query times, merge queue, replication lag
- **Redpanda Health** — consumer lag, throughput, partition leaders
- **Agent Fleet** — agent status, collection mode, heartbeat map
- **Security Posture** — open alerts by severity, MITRE heatmap, top assets

Access: `https://grafana.kron.{org}.internal` (same SSO as KRON)

### Critical Alerts (Prometheus → KRON itself)
```yaml
# kron fires on its own Prometheus metrics:
- alert: KRONIngestionStopped
  expr: rate(kron_events_ingested_total[5m]) == 0
  for: 5m

- alert: KRONConsumerLagHigh
  expr: kron_redpanda_consumer_lag > 100000
  for: 10m

- alert: KRONClickHouseSlowQueries
  expr: histogram_quantile(0.99, kron_clickhouse_query_duration_seconds) > 10
  for: 5m

- alert: KRONAgentsDark
  expr: kron_agents_dark_count > 0
  for: 2m
```

## Backup and Recovery

### Automated Backups
```bash
# Configure in values.yaml:
backup:
  enabled: true
  schedule: "0 2 * * *"  # nightly at 2am
  destination: "s3://kron-backups/"  # MinIO on-prem
  retention_days: 90
  encryption: true
```

### Manual Backup
```bash
kron-ctl backup create --include-audit-log --output /backup/kron-$(date +%Y%m%d).tar.gz
```

### Restore
```bash
# Full restore from backup:
kron-ctl restore --from /backup/kron-20260101.tar.gz

# Selective table restore:
kron-ctl restore --from /backup/kron-20260101.tar.gz --tables events --tenant acmebank --from-date 2026-01-01
```

See Runbooks in the security documentation for disaster recovery procedures.

## Uninstall

```bash
# Nano:
sudo kron-ctl uninstall --purge-data  # irreversible — deletes all data

# Standard/Enterprise:
helm uninstall kron -n kron
kubectl delete namespace kron
# Data volumes are NOT deleted automatically — must be deleted manually:
kubectl delete pvc --all -n kron  # WARNING: irreversible
```
